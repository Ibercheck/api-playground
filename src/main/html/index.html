How to start
  1. Exchange your partner secret with an OAuth2 access_token /oauth
  2. Now use the access_token returned and fill the top bar access_token form field
  3. You are ready to play your experiments
Do your first report request
Reports are asynchronous transactions. This means it's necessary to register a request. After registering your request you will be posted about the progress via webhooks.
  1. Register a new user performing a POST operation over /user endpoint
  2. Next register a new sale for any product present in your product catalog. This sale will be linked to the user account and will appear in his/her dashboard. Send a POST message to the link "sale" present into the user object, usually /user/{user_id}/sale
  3. Now the user has credits in his/her inventory for creating a report request. Send a POST message to the link "report-request" present in the sale object created before, usually /sale/{sale_id}/report-request
  4. If your report request was a "check_a_tercero" you are done. We will notify you with the final result. If you created an "autocheck" then we need more data for proper user identification.
    Please fill the user's name, surname, mobile phone and national ID of the user with PATCH operation over the user object self link, usually /user/{user_id}
  5. We need a proof of the user identity. You have to upload a photo of his/her current, not expired, national Id card. You can do it through the "authorization_document" link present in the report-request object, usually /report-request/{report_request_id}/authorize/document
  6. Finally, complete the authentication process sending an empty POST message to the authorization link present in the report request object, usually /report-request/{report_request_id}/authorize/
  7. Once the authentication step is completed we need to collect the user authorization for checking his/her personal data according to the Spanish protection data laws.
    A new link in your report request object, called authorization_online_sign, is generated upon completion of the authorization process. You have to send this link to the user's browser so he/she can complete the process.
